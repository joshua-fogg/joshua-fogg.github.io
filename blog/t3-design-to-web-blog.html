What is a responsive site, and why is responsiveness important.

A responsive site is designed to react to the size of the browser and screen being used. Scalabel objects and screen ratio's or percentages can be used to do this.

Screen size parameters can be used to choose between different layouts and interfaces that keep the users experience feeling optimal. Overall ensuring an easy experience and accessibility across the website.


Mobile first design: what and why is it important.

In 2016 mobile devices such as mobiles and tablets became more common for accessing the internet than desktops and laptops (Guardian 2016 ). With this in mind it is no wonder that the web development sector is moving to a mobile first design plan.

As there are now more mobile users it makes sense to have either a responsive display or to deisgn a site with the largest amount of users in mind, the mobile community. Especially when approaching site development from a commercial aspect.


What frameworks are, and their pros and cons.

Frameworks give a standardised platform for web development, effectively defining the toolbox for us developers. Frameworsk also gives a coherency to our work, allowing for better communication and easier learning between developers. See below for a quick summary of the pro's and con's around frameworks.

Pros: Cons:
Higher work efficiency Creative restrictions and limitations
Platform browser compatibilty Slower learning curve
Helps keep code clean and tidy Coding becomes public and copyable

What is a wireframe and why use it:

Wireframes are a basic line drawing of the layout of box elements within a page.

  • They enable communication of draft ideas.
  • Allow for quick drafting of layouts.
  • Allow a means to visually brainstorm.
